# Mermaid 流程图查看器需求设计

## 1. 背景与目标

当前编辑器会在 Markdown 预览区内直接渲染 Mermaid 图表。预览正文最大宽度约为 1020px，大型、超宽或节点密集的流程图会被压缩，用户难以看清文字和完整结构。

本特性为每个成功渲染的 Mermaid 图表提供独立查看器，使用户能够自由缩放和平移大型图表，在同一篇文档的多张图表之间切换，并将单张完整图表导出为 PNG 或 SVG。

本次交付适用于：

- 在线版 `markdown-editor.html`
- 由 `build-offline.js` 生成的离线版

Electron 桌面应用的整体方案尚未确定，不在本次验收范围内；本特性不得修改 `electron/` 目录。

## 2. 范围

### 2.1 包含内容

- Mermaid 图表的悬浮“查看大图”入口
- 双击图表打开查看器
- 覆盖页面的全屏模态查看器
- 鼠标、键盘和触控操作
- 缩放、平移、适应窗口、100% 显示
- 大图自适应 minimap
- 多图上一张、下一张浏览
- 一键复制当前图表的原始 Mermaid 源码
- 完整图表的 PNG、SVG 导出
- 透明、白色、自定义颜色背景
- 浅色和深色主题适配
- 在线和离线 Web 版验证

### 2.2 不包含内容

- 编辑 Mermaid 源码或节点
- 导出当前裁剪视口
- 为非 Mermaid 图片提供通用查看器
- 浏览器原生全屏模式
- 循环切换首尾图表
- 将查看器拆分为新的页面或 Electron 窗口
- Electron 桌面端适配与验收

## 3. 技术方向

采用无新增依赖的原生 SVG 查看器。

查看器复用 Mermaid 已生成的 SVG，通过 Pointer Events、触摸手势和变换矩阵完成缩放与平移。导出和 minimap 都以未应用查看变换的原始 SVG 副本为数据源。

选择该方案的原因：

- 保持项目“单 HTML 核心”的现有架构
- 不增加 CDN、离线资源和依赖维护成本
- 在线和离线 Web 版行为一致
- SVG 导出链路直接，且不会受查看器当前状态影响
- 可以精确实现自适应 minimap 和自定义背景导出

## 4. 架构与组件

在 `markdown-editor.html` 内新增独立的 `MermaidViewer` 模块。模块只依赖预览区已生成的 Mermaid SVG、现有 Toast、现有 Storage 包装和浏览器标准 API。

### 4.1 图表发现与入口

Mermaid 异步渲染完成后，扫描预览区中有效的 `.mermaid svg`：

- 为每张图建立稳定的当前渲染序号
- 在外层增加查看容器和悬浮按钮
- 绑定双击打开行为
- Mermaid 渲染失败且没有有效 SVG 时不增加入口
- 重复执行增强逻辑时不得重复创建按钮或事件

图表重新渲染后，旧 DOM 引用可能失效。打开查看器或切换图表时必须重新从当前预览区读取图表列表。

### 4.2 查看器外壳

查看器是覆盖当前页面的全屏模态层，不调用浏览器 Fullscreen API。它包括：

- 顶部工具栏
- 中央 SVG 画布
- 左右图表导航按钮
- 当前图表计数
- 条件显示的 minimap
- 导出设置面板
- 关闭入口

查看器打开期间屏蔽页面背景滚动，并暂停由预览区交互触发的滚动同步；关闭后恢复原状态。

### 4.3 变换控制器

控制器维护以下会话状态：

- 当前图表索引
- 当前缩放比例
- X/Y 平移量
- 当前指针集合
- 拖动或捏合状态

视图变换只应用于查看器中的 SVG 副本，不修改预览区 SVG，不修改 Markdown 内容。

### 4.4 Minimap

minimap 使用目标 SVG 的轻量副本展示完整图表，并用视口框表示主画布当前可见区域。

当图表在 100% 比例下任一方向超过主画布对应尺寸 1.5 倍时自动显示。小图不显示。minimap 不提供手动关闭入口，满足阈值时始终保留。

### 4.5 导出器

导出器始终重新克隆未缩放的完整 SVG，不使用查看器当前的缩放、平移或裁剪状态。

导出设置通过现有 Storage 保存：

- 最近使用的背景模式：透明、白色、自定义颜色
- 最近使用的自定义颜色值

临时 Blob URL、Canvas 和导出 DOM 必须在完成或失败后清理。

## 5. 数据流

```text
Markdown 内容
  -> marked 与 DOMPurify
  -> Mermaid 异步渲染 SVG
  -> 扫描有效 SVG 并注入查看入口
  -> 用户打开指定图表
  -> 克隆原始 SVG 到查看器
  -> 查看器执行缩放、平移和 minimap 同步
  -> 用户发起导出
  -> 再次克隆未变换的完整 SVG
  -> 内联必要样式并应用背景
  -> 下载 SVG，或栅格化为高清 PNG 后下载
```

## 6. 交互设计

### 6.1 打开与关闭

- 桌面端鼠标移入图表时，在右上角显示“查看大图”按钮
- 触控设备始终提供可点击的查看按钮
- 双击图表的非链接区域打开查看器
- Mermaid 图内原有链接保持可点击，链接交互不得被双击处理拦截
- 使用右上角按钮、`Esc` 或点击模态遮罩关闭
- 在导出设置或其他浮层打开时，点击浮层本身不得关闭查看器

### 6.2 工具栏

顶部工具栏提供：

- 图表名称，缺少标题时显示“Mermaid 图表 N”
- 上一张
- 下一张
- 缩小
- 当前缩放比例
- 放大
- 适应窗口
- 100%
- 导出
- 复制代码
- 关闭

查看器显示“图表 N / 总数”。第一张禁用“上一张”，最后一张禁用“下一张”，不循环切换。

### 6.3 缩放与平移

- 打开或切换图表后默认执行“适应窗口”并居中
- 支持 10% 至 500% 缩放
- 缩放按钮使用固定步长
- 鼠标滚轮以指针位置为缩放中心
- 触控双指捏合以双指中心为缩放中心
- 鼠标按住拖动或触屏单指拖动进行平移
- 缩放和平移应保持连续，不出现明显跳变
- `+`、`-` 控制缩放，`0` 恢复 100%，`F` 适应窗口
- 输入焦点位于按钮、颜色输入框或文本输入框时，不响应单键快捷键
- 双击查看器画布不承担额外缩放动作，避免和打开图表的双击含义冲突

### 6.4 多图导航

- 左右按钮和键盘 `Left`/`Right` 切换图表
- 切换后重新执行“适应窗口”，不继承上一张图的变换状态
- 切换时重新读取预览区当前有效 SVG，处理 Markdown 重渲染导致的 DOM 失效
- 当前图表在新列表中不存在时，选择最接近的有效索引；没有有效图表时关闭并提示

### 6.5 Minimap 交互

- minimap 与主画布的缩放、平移实时同步
- 点击 minimap 将主视口中心移动到对应位置
- 拖动 minimap 视口框可连续导航
- 不提供关闭按钮；仅由图表原始尺寸阈值控制显示与隐藏

### 6.6 复制 Mermaid 源码

- “复制代码”按钮位于顶部工具栏的“导出”按钮左侧
- 复制当前图表对应的原始 Mermaid 文本，不包含 Markdown 三反引号代码围栏
- 渲染 Mermaid 前将源码与生成容器关联，切换图表时同步切换源码
- 优先使用 `navigator.clipboard.writeText()`
- Clipboard API 不可用、权限受限或页面不处于安全上下文时，使用临时文本框和 `document.execCommand('copy')` 回退
- 成功后提示“Mermaid 代码已复制”，失败时提示“复制失败，请手动复制”

## 7. 导出设计

### 7.1 通用规则

- 格式可选择 PNG 或 SVG
- 背景可选择透明、白色或自定义颜色
- 自定义颜色使用颜色选择器，并显示颜色值
- 记住最近一次背景模式与自定义颜色
- 导出范围固定为完整图表
- 不包含查看器按钮、minimap、缩放比例或其他界面元素
- 文件名格式：`Markdown文件名-mermaid-N.png` 或 `.svg`
- 文件名必须移除不适用于本地文件系统的字符

### 7.2 SVG 导出

- 克隆完整原始 SVG
- 补齐 XML 命名空间、明确宽高和 `viewBox`
- 内联独立打开所需的样式
- 透明背景不插入背景元素
- 白色或自定义背景在最底层插入覆盖完整 `viewBox` 的 `<rect>`
- 导出结果独立打开后应保留文字、连线、颜色、字体回退和背景

### 7.3 PNG 导出

- 默认以 SVG `viewBox` 对应完整尺寸的 2 倍栅格化
- 保持 SVG 宽高比
- 透明背景保留 Alpha 通道
- 白色或自定义背景先填充 Canvas，再绘制 SVG
- 最长边限制为 16384px，总像素限制为 64,000,000px
- 超限时等比例降低倍率，并提示最终导出尺寸
- PNG 转换不得改变查看器当前视图状态

如浏览器不能直接正确栅格化包含 HTML 标签的 Mermaid SVG，应以内联样式后的 SVG 为首选输入，并提供基于离屏 DOM/html2canvas 的兼容回退；两条路径都必须导出完整图表。

## 8. 主题与可访问性

- 查看器外壳适配现有浅色和深色模式
- 图表默认保持打开时的 Mermaid 渲染样式
- 工具栏按钮提供中文提示、可访问名称和清晰焦点状态
- 所有核心动作均可通过键盘完成
- 模态打开时将焦点移入查看器并限制焦点循环
- 关闭后将焦点恢复到触发查看器的按钮或图表容器
- 背景选择为透明且可能造成低对比度时给出非阻塞提示，用户仍可继续导出
- 尊重 `prefers-reduced-motion`，减少非必要动画

## 9. 错误处理与资源清理

- Mermaid 没有生成有效 SVG：不显示入口
- 打开时目标 SVG 已失效：刷新图表列表并尝试恢复当前索引
- 文档中已没有有效图表：提示后关闭查看器
- SVG 序列化、PNG 栅格化或下载失败：显示明确错误 Toast，不关闭查看器
- 超大 PNG：安全降低导出倍率，不尝试创建超出限制的 Canvas
- 导出中的重复点击：禁用导出动作并显示进度状态
- 找不到当前图表源码：不写入剪贴板，显示明确错误提示
- 剪贴板主路径失败：自动尝试临时文本框回退；两条路径都失败后再提示失败
- 关闭查看器：释放指针捕获、移除会话级监听器、撤销 Blob URL、清空临时 DOM 和 Canvas
- 任一错误不得修改 Markdown 源文档或预览区原始 SVG

## 10. 与现有渲染链路的集成

当前 `render()` 调用 `renderMermaid()` 后，通过 `requestAnimationFrame` 构建滚动映射。Mermaid 11 的 `mermaid.run()` 返回 Promise，因此增强入口和后续高度计算应在 Mermaid 渲染真正完成后执行，避免扫描到尚未生成 SVG 的节点。

实现时应将 Mermaid 后处理设计为幂等操作，并确保：

- 入口增强不会重复包裹 DOM
- 滚动映射在 Mermaid SVG 和入口容器稳定后构建
- 查看器 DOM 不参与正文滚动映射
- 整篇文档的 PDF、PNG、HTML 导出不包含悬浮查看按钮
- 打印样式隐藏所有查看器入口
- 只修改在线源文件，离线 HTML 由构建脚本重新生成

## 11. 验证策略

项目当前没有测试框架。验证由可独立测试的纯函数、浏览器交互检查和在线/离线 Web 构建检查组成。

### 11.1 图表样本

- 小型流程图
- 横向超宽流程图
- 纵向超高流程图
- 节点密集流程图
- 时序图、类图、状态图、ER 图等非流程图 Mermaid 类型
- 单图和多图文档
- 同时包含有效和无效 Mermaid 代码的文档
- 带链接或 HTML 标签的 Mermaid 图

### 11.2 交互验证

- 悬浮按钮和双击均可打开正确图表
- 链接点击不被打开逻辑破坏
- 打开、关闭和焦点恢复正确
- 鼠标滚轮、按钮、键盘、拖拽操作正确
- 单指拖动和双指捏合正确
- 10% 与 500% 边界生效
- 适应窗口和 100% 行为正确
- 多图切换、首尾禁用和计数正确
- minimap 自动显示阈值、定位和拖动正确，且没有关闭入口
- 当前图表原始 Mermaid 源码可以复制，内容不包含 Markdown 代码围栏
- Clipboard API 不可用时回退复制路径可执行并清理临时文本框
- 重复打开和关闭不累积事件监听器或临时节点

### 11.3 导出验证

- PNG 和 SVG 均支持透明、白色、自定义颜色
- 导出始终包含完整图表
- 文件名、尺寸和宽高比正确
- SVG 可独立打开且样式完整
- PNG 透明通道和自定义颜色正确
- 超大图安全降低倍率并提示
- 导出后查看器变换状态不变
- 导出失败后仍可继续操作或重试

### 11.4 Web 回归验证

- `markdown-editor.html` 在线版
- `npm run build:offline` 生成的离线版
- 浅色、深色主题
- 现有 Markdown、KaTeX、高亮和 Mermaid 渲染
- 编辑器与预览区滚动同步
- 整篇 PDF、PNG、HTML 导出
- 打印输出不包含查看入口或查看器 UI

## 12. 验收标准

1. 任意成功渲染的 Mermaid 图都可通过悬浮按钮或双击打开。
2. 大型图可在 10% 至 500% 范围内平滑缩放，并可拖动查看任意区域。
3. 打开和切换图表后能自动适应窗口；用户可恢复 100%。
4. 超出阈值的大图自动显示可交互 minimap，小图不显示。
5. 用户可在同一查看器中通过按钮或方向键浏览文档内全部有效 Mermaid 图。
6. 用户可导出完整 PNG 或 SVG，并选择透明、白色或自定义背景。
7. 导出结果不受当前查看器缩放和平移影响，也不包含查看器 UI。
8. 超大 PNG 不导致页面崩溃，系统会安全降低导出倍率并反馈结果。
9. 鼠标、键盘和触控的核心操作均可用。
10. 在线和离线 Web 版行为一致。
11. 功能不修改 Markdown 内容，不破坏滚动同步和现有整篇文档导出。
12. 用户可复制当前图表的原始 Mermaid 源码，切换图表后复制内容同步变化。

## 13. 实施边界

本设计可作为单个实现计划交付。实现顺序应优先保证图表发现、模态查看、基础缩放平移和 SVG 导出，再完成 PNG、minimap、多图切换、触控细节与在线/离线 Web 回归。

任何新增第三方依赖、改变离线构建模式、修改 Mermaid 源码语义或扩展为通用图片查看器的需求，都视为本规格之外的变更，需要重新评估。
